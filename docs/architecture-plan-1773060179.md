## Анализ задачи: Нужно создать телеграм бота, который будет на вход получать картинку с изображением джиу-джитсу. В ответ надо отправить 3 фразы на англ. Первая фраза в разговорном смешном варианте, вторая фраза в профессиональном, третья фраза должна описывать, что происходит на картинке. Стек давай выберем PHP, а остальное по усмотрению.

- Потенциально хрупкие зоны (на старте проекта): обработка больших изображений, лимиты Telegram/OpenAI, сетевые таймауты, повторная доставка webhook-событий.

### Риски и зависимости
- `HIGH` Неопределённость AI-провайдера и модели vision: без фиксированного API и SLA может измениться формат ответа и латентность.
- `HIGH` Риск некорректной обработки webhook retry от Telegram: без идемпотентности бот может отвечать дублирующими сообщениями.
- `HIGH` Нарушение требований к формату ответа (ровно 3 фразы, нужные стили, английский язык) при слабом prompt-контракте.
- `MEDIUM` Ограничения размера/качества изображения: часть изображений может не пройти скачивание или анализ.
- `MEDIUM` Отсутствие очереди/асинхронности: при долгом распознавании возможны timeout и деградация UX.
- `MEDIUM` Секреты (Telegram token, AI API key): риск утечки без централизованной конфигурации и маскирования логов.
- `LOW` Эволюция prompt-шаблонов может ухудшить качество без regression-тестов на примерах.

Зависимости между компонентами:
- Контракты домена блокируют адаптеры Telegram и AI.
- Telegram adapter и AI adapter могут разрабатываться параллельно после фиксации контрактов.
- Оркестрация use-case зависит от готовности обоих адаптеров.
- Наблюдаемость, отказоустойчивость и финальные тесты зависят от сквозного пайплайна.
